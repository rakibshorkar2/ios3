Here’s a practical AI-agent prompt specifically for implementing a Flutter-only torrent engine first (no native libtorrent yet) and integrating it into your existing Downloads tab.

### Prompt for AI Agent

Add a Flutter-only BitTorrent engine to my existing Flutter iOS app for testing purposes. Do not use native Swift/libtorrent yet. Use a pure Dart torrent library and integrate it into the current Downloads tab.

Requirements

* Create a TorrentEngineService in Dart.

* Use a maintained Dart BitTorrent package capable of handling magnet links, metadata retrieval, peer connections, and file downloading.

* Do not modify existing HTTP download logic.

* Integrate torrent downloads into the existing download queue and download card UI.

Magnet Support

* Detect `magnet:?xt=urn:btih:` links in the existing “New Download” dialog.

* Parse and validate the magnet URI.

* Extract torrent name, info hash, and trackers.

* Start metadata retrieval asynchronously.

Metadata Retrieval

* Show live status: “Connecting to peers…”, “Fetching metadata…”, “Metadata X%”.

* Wait up to 120 seconds before failing.

* If metadata is received, show a confirmation dialog containing torrent name, total size, file count, and file list.

* Allow selecting all files or individual files.

Downloading

* Start downloading selected files into the app’s existing download directory.

* Support pause, resume, cancel, and delete.

* Persist torrent state locally so unfinished downloads can be restored after app restart.

* Enable sequential download mode for media files.

Progress UI

Each torrent item in the Downloads tab should display:

* Progress percentage

* Downloaded size / total size

* Download speed

* Upload speed

* ETA

* Peer count

* Seeder count

* Current state (metadata, downloading, paused, seeding, completed, error)

Architecture

* Create `TorrentEngineService` for torrent operations.

* Create `TorrentTaskModel` for state management.

* Create `TorrentRepository` for persistence.

* Expose progress through `Stream`s or the project’s existing state-management solution.

* Keep the torrent layer modular so it can later be replaced with a native libtorrent backend without changing the UI.

Error Handling

Show friendly messages for:

* Invalid magnet link

* Metadata timeout

* No peers found

* Storage full

* Network unavailable

Code Quality

* Use asynchronous APIs only; never block the UI thread.

* Add detailed logging for metadata retrieval, peer discovery, and download progress.

* Preserve the current app theme and Downloads tab layout.

* Write production-quality, null-safe Dart code compatible with the current Flutter version.

### One extra line to add if the agent asks which package to use

Prefer a pure Dart BitTorrent package that supports magnet metadata retrieval, peer discovery, and file downloading on iOS. If multiple packages are available, choose the most actively maintained one and explain why.

This prompt is intentionally scoped for “Flutter-only first” so the agent won’t immediately jump into complex Swift/libtorrent setup. Once this prototype works, you can later swap `TorrentEngineService` with a native libtorrent implementation while keeping the same UI and state management.
